# Code of Conduct

## Our Standards

We want tatsumaki to be a respectful and productive project.

Examples of behavior that helps the project:

- Be direct, kind, and constructive.
- Assume good intent, but clarify disagreements with concrete examples.
- Keep discussions focused on the work and user impact.
- Respect maintainers' time by keeping issues and pull requests actionable.

Examples of behavior that is not acceptable:

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeated off-topic or disruptive comments.
- Publicly disclosing security vulnerabilities before a coordinated fix.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants when behavior harms the project or its community.

If you see behavior that violates this Code of Conduct, contact the maintainers privately. Security vulnerabilities should be reported through the process in [SECURITY.md](SECURITY.md), not through public issues.
