# Documentation Management

## 目的

入口ドキュメントをポインタとして保ち、詳細ルールは参照先へ集約する。

## 新規ルール追加時の判断手順

1. 追加内容が「エージェントの実行開始に必須」か判定する
2. 必須なら `AGENTS.md` / `CLAUDE.md` に短いポインタを追加する
3. 必須でなければ既存の参照先ドキュメントへ追記する
4. 既存ドキュメントの責務に収まらない場合のみ、新規ドキュメントを追加する
5. `docs/knowledge-index.md` に責務・更新責任・最終更新日を登録する
6. `bash scripts/check-docs-links.sh` で死リンクと未参照を検証する

## 更新責任

- ルール本文の更新者は同じPRでポインタと知識索引も更新する
- 入口ドキュメントには詳細本文を書かず、参照リンクだけを維持する
