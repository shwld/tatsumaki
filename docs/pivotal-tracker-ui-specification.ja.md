# Pivotal Tracker UI 仕様分析

このドキュメントは、Pivotal TrackerのUI/UXを詳細に分析し、ページ構造、主要コンポーネント、およびステート駆動型のワークフローロジックをまとめたものです。

---

## 1. 全体ページ構成
Pivotal Trackerは、**高密度なマルチパネル・ワークスペース**レイアウトを採用しています。画面を切り替えることなく、プロジェクトの複数の状態を同時に確認・操作できるのが特徴です。

### A. グローバルナビゲーションバー（上部）
- **プロジェクトスイッチャー**: プロジェクトやワークスペースを切り替えるドロップダウン。
- **検索バー**: 強力なクエリ言語（例：`state:started`）をサポートする検索フィールド。検索結果は新しいパネルとして開かれます。
- **プロジェクト進捗（ベロシティ）**: 現在のプロジェクトのベロシティとイテレーションの進捗状況を表示。
- **設定/プロファイル**: ユーザー設定、通知、プロジェクト設定へのアクセス。

### B. プロジェクトサイドバー（左側）
サイドバーは、ワークスペースのパネル配置を制御するメインコントローラーです。
- **My Work**: 自分に割り当てられたストーリーのみを表示するパネル。
- **Current / Backlog**: 現在のイテレーションと、優先順位付けされた今後の作業パネル。
- **Icebox**: 未計画・未優先のストーリーを保管するパネル。
- **Done**: 完了（Accepted）したストーリーの履歴。過去のイテレーションごとにグループ化されます。
- **Epics**: 大規模な機能やテーマのリスト。
- **Labels**: プロジェクトで使用されている全ラベルのリスト。クリックするとフィルタリングとして機能します。
- **Analytics**: バーンアップチャート、累積フロー図、サイクルタイムなどへのリンク。

### C. ワークスペース（メインエリア）
- **水平スクロール**: パネルは横並びに配置されます。画面幅を超える数のパネルが開いている場合、水平にスクロールして移動します。
- **パネルのリサイズ**: 垂直カラム（パネル）は、必要に応じてリサイズや折り畳みが可能です。

---

## 2. パネル（構成要素）
パネルはストーリーカードの垂直リストです。ストーリーが「どこにあるか」によってその状態を表します。

| パネル名 | ロジック / 内容 |
| :--- | :--- |
| **Icebox** | イテレーションに割り当てられていないストーリー。新しいアイデアや下書きの置き場。 |
| **Backlog** | 優先順位が付けられ、*将来*のイテレーションにスケジュールされたストーリー。 |
| **Current** | *現在*のイテレーションにスケジュールされているストーリー。容量はチームの**ベロシティ**によって自動決定されます。 |
| **Done** | **Accepted（承認）**されたストーリー。承認されたイテレーションごとに表示されます。 |
| **My Work** | 自分がオーナーとなっているストーリーを、プロジェクト横断（ワークスペース時）またはプロジェクト単位で集約したビュー。 |

---

## 3. ストーリーカード：最小構成単位
ストーリーカードは情報の中心です。**コンパクト表示**と**展開表示**の2つの状態を持ちます。

### A. コンパクト表示（リスト形式）
| コンポーネント | 操作 | 説明 |
| :--- | :--- | :--- |
| **展開矢印** | クリック | コンパクト表示と展開表示を切り替えます。 |
| **種別アイコン** | 表示 | 星（Feature）、虫（Bug）、歯車（Chore）、旗（Release）で種類を判別。 |
| **見積もり（ポイント）** | 表示 | 0, 1, 2, 3などの数値を表示。未見積もりは「-」表示。 |
| **ステートボタン** | クリック | **Start**, **Finish**, **Deliver**, **Accept**, **Reject** など、状態に応じた動的なボタン。 |
| **タイトル** | 表示 | ストーリーの概要。 |
| **ラベル** | 表示 | 小さなカラータグ。クリックするとそのラベルでフィルタリングされます。 |
| **オーナーイニシャル** | 表示 | 担当者のイニシャルが入った円形アイコン。 |

### B. 展開表示（詳細エディタ）
展開すると、パネル内で**インライン**に以下のコントロールが表示されます。

| セクション | コンポーネント | 説明 |
| :--- | :--- | :--- |
| **タイトル** | テキストエリア | 編集可能なタイトルフィールド。 |
| **見積もり（Point）** | ポイントボタン | プロジェクトの設定に基づいたポイント（線形、フィボナッチ等）を選択。 |
| **説明** | Markdownエディタ | 要件や受け入れ条件を記載するリッチテキストエリア。 |
| **タスク** | チェックリスト | 実装に必要なサブタスク。開発者が完了を確認するために使用。 |
| **コメント** | スレッドリスト | `@メンション` やファイル添付が可能な議論エリア。 |
| **アクティビティ** | ログ | 状態変更、オーナー割り当て、編集履歴の表示。 |
| **Requester** | 選択 | 要件の「オーナー」（通常、承認/却下を行う人）。 |
| **Owners** | 複数選択 | 実装に責任を持つ1人以上の担当者。 |
| **Followers** | リスト | ストーリーの通知を受け取るユーザー。 |
| **ラベル** | テキスト入力 | タグの追加・削除。 |
| **Blockers** | リンク | 依存関係にある他のストーリーへのリンク。 |

## 4. ステートマシンとワークフローボタン
Pivotal Trackerの核心は、厳格な状態遷移ロジックにあります。

### Feature（機能）および Bug（バグ）のワークフロー
1. **Unstarted** (Backlog/Icebox内)
   - ボタン: `Start`
2. **Started**
   - ボタン: `Finish`
3. **Finished** (実装完了)
   - ボタン: `Deliver`
4. **Delivered** (検証可能状態)
   - ボタン: `Accept` (緑) / `Reject` (赤)
5. **Accepted** -> **Done**パネルへ移動
   - *このイテレーションでの作業は完了。*
6. **Rejected**
   - ボタン: `Restart` -> **Started**状態へ戻る。

### Chore（雑務）のワークフロー
- **Unstarted** -> `Start`
- **Started** -> `Finish`
- **Accepted**（Choresはユーザー価値ではなく技術的な必要性を表すため、DeliverやRejectの工程をスキップします）。

## 5. 自動化ロジック：イテレーションとベロシティ
ユーザーが手動で「スプリント」を作成するのではなく、Trackerが自動的に管理します。
- **ベロシティ計算**: 直近3イテレーションの完了ポイントの平均。
- **イテレーションマーカー**: Backlogパネル内に引かれる水平線。ベロシティが10なら、10ポイントごとに線が引かれ、どの週に完了するかを予測します。
- **オーバーキャパシティ**: Currentイテレーションのポイントがベロシティを超えると、ストーリーは自動的にBacklogの次のイテレーションマーカーへ押し出されます。

## 7. プロジェクト設定（Project Settings）
プロジェクトの動作を決定する核心的な設定項目です。

### A. 基本設定（General Settings）
| 項目 | 説明 |
| :--- | :--- |
| **Project Name** | プロジェクトの名称。 |
| **Project Description** | プロジェクトの概要。 |
| **Public / Private** | 公開・非公開の切り替え。Publicの場合、ログインしていないユーザーも閲覧可能（設定による）。 |
| **Iteration Length** | 1イテレーションの長さ（通常は1週間、2週間など）。 |
| **Week Start Day** | イテレーションの開始曜日（通常は月曜日）。 |
| **Time Zone** | プロジェクトのタイムゾーン。 |

### B. 予測とベロシティ（Velocity & Forecasting）
| 項目 | 説明 |
| :--- | :--- |
| **Velocity Strategy** | 直近何イテレーション分の平均をベロシティとして使用するか（例：直近3回）。 |
| **Initial Velocity** | プロジェクト開始時の初期ベロシティ設定。 |
| **Point Scale** | ストーリーの見積もりに使用するスケール（Linear, Fibonacci, Powers of 2, またはカスタム）。 |

### C. 見積もり対象の設定（Estimation Settings）
| 項目 | 説明 |
| :--- | :--- |
| **Estimate Bugs** | バグ（Bug）にポイントを付けるかどうか。 |
| **Estimate Chores** | 雑務（Chore）にポイントを付けるかどうか。 |

## 8. メンバーと権限管理（Members & Roles）
プロジェクトに参加するユーザーの管理と、操作権限の定義です。

### A. ユーザーロール（Roles）
| ロール | 権限概要 |
| :--- | :--- |
| **Project Owner** | 全ての設定変更、メンバーの追加・削除、プロジェクトの削除が可能。 |
| **Member** | ストーリーの作成・編集、ステータスの変更、コメントの投稿が可能。 |
| **Viewer** | 閲覧のみ可能。編集やステータス変更は不可。 |

### B. 招待とアクセス制御
- **Email Invitation**: メールアドレスによるメンバー招待。
- **Account Admin**: アカウントレベル（複数プロジェクトを束ねる単位）でのユーザー管理。

---

## 9. 外部連携と通知（Integrations & Notifications）
### A. インテグレーション
- **GitHub / GitLab**: コミットメッセージに `[#STORY_ID]` を含めることで、ストーリーとコードを紐付け。
- **Slack**: ストーリーの作成やステータス変更をチャンネルに通知。
- **Webhooks**: 状態変更時に外部URLへJSONデータを送信。

### B. 通知設定（My Notifications）
- **Email Notifications**: 自分がアサインされた時、コメントされた時などのメール通知のON/OFF。
- **In-app Notifications**: ブラウザ上での通知表示。

---

## 10. インポートとエクスポート（Data Management）
- **CSV Import**: 大量のストーリーを一括で登録。
- **CSV Export**: プロジェクト全データをバックアップや分析用に書き出し。
- **API Access**: APIトークン（Token）の発行。外部ツールやスクリプトからの操作に使用。

---

## 11. 分析画面（Analytics & Charts）
チームのパフォーマンスとプロジェクトの進捗を視覚化するための統計ツールです。

### A. 主要なチャート（Core Charts）
| チャート名 | 分析の目的 |
| :--- | :--- |
| **Velocity Chart** | イテレーションごとの完了ポイントを表示。過去3〜12週の平均（ベロシティ）を確認し、将来の予測精度を高めます。 |
| **Burn-up Chart** | 全体のスコープ（Total Points）に対して、完了したポイント（Accepted Points）の推移を表示。リリース予定日への到達度を視覚化します。 |
| **Cumulative Flow** | ストーリーの各状態（Unstarted, Started, Finished, Delivered, Accepted）の累積量を面グラフで表示。ボトルネック（例：Deliveredで滞留している等）を特定します。 |
| **Cycle Time** | ストーリーが「Started」から「Accepted」になるまでにかかった平均時間を計測。作業のスピードと効率を分析します。 |
| **Rejection Rate** | デリバリーされたストーリーのうち、何%が「Rejected」されたかを表示。要件定義の質や開発の品質を測る指標となります。 |

### B. エピック進捗（Epic Progress）
- **Epic Burn-up**: 特定のエピックに紐付くストーリーの完了状況を追跡。大規模な機能のリリース予測に使用します。

---

## 12. その他のグローバル機能（Global Features）
- **ドラッグ＆ドロップ**: ストーリーをIceboxからBacklogへ移動することで「スケジューリング」されます。Backlog内での並び替えは優先順位の変更を意味します。
- **一括操作（Bulk Actions）**: チェックボックスで複数のカードを選択し、状態、ラベル、オーナーを一括変更できます。
- **リアルタイム同期**: 1人のユーザーが行った変更は、同じプロジェクトを見ている全ユーザーの画面に即座に反映されます。
- **ショートカットキー**: `n`（新規ストーリー）、`s`（検索）、`j`/`k`（カード移動）など、キーボードのみで高速に操作するためのショートカット。
