# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

For the public repository, report vulnerabilities using GitHub's private vulnerability reporting feature from the repository's Security advisories page.

When reporting a vulnerability, include as much detail as possible:

- Affected component
- Steps to reproduce
- Potential impact
- Suggested fix, if known

We will acknowledge valid reports as soon as practical and coordinate disclosure after a fix is available.

## Supported Versions

tatsumaki is currently pre-1.0. Security fixes are provided for the latest commit on the main branch unless otherwise stated.
