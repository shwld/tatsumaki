# データアクセス

- D1のテーブル管理は `apps/web/migrations/` の migration を唯一の変更経路とする。
- migration のスキーマ定義では、主キーにソート可能な文字列ID（ULIDなど）を採用する。
- アプリケーションコードで `CREATE TABLE` などのスキーマ作成SQLを実行しない。
- 永続化処理は生SQLを増やさず、原則 `drizzle-orm` を利用する。
- 検索・並び替え・ページングをフロントエンド（クライアント）だけで実装することを禁止する。データ量の増加に伴いパフォーマンスが劣化するため、これらの処理はSQL（サーバー側クエリ）に寄せる。フロントエンドは結果の表示に徹し、フィルタ条件やソート順、ページ指定をサーバーに渡す設計とする。
